# 页面设计文档：主页（Desktop-first）

## 1. Layout
- 栅格：12 列 CSS Grid（内容区），左右留白自动；主容器 `max-width: 1200px`（大屏可 1280px），`padding: 0 24px`。
- 结构：顶部导航（sticky 可选）+ 多段纵向 section 堆叠；段与段之间用一致的垂直节奏（48/72/96px）。
- 对齐：核心信息左对齐（更专业）；关键数据与徽章可居中或右侧对齐作为平衡。
- 响应（以桌面为主）：
  - ≥1200px：三列价值支柱、三步路径横向卡片
  - 768–1199px：两列布局
  - <768px：单列堆叠、按钮全宽、导航折叠为抽屉

## 2. Meta Information
- Title：专业 / 可审计 / 安全优先｜让关键系统更可信
- Description：用可验证的审计证据与安全优先的工程方法，帮助你在最短路径内完成接入、验证与上线运行。
- Open Graph：
  - og:title：专业 / 可审计 / 安全优先
  - og:description：三步路径：接入 → 可审计验证 → 安全上线
  - og:type：website

## 3. Global Styles（Design Tokens）
- 颜色
  - Background：`#0B1220`（深海军蓝，专业与安全感）
  - Surface：`#111B2E`（卡片底）
  - Text Primary：`#EAF0FF`
  - Text Secondary：`#A9B4D0`
  - Accent：`#7C5CFF`（紫蓝强调，科技但不轻浮）
  - Success/Trust：`#2FE4AB`
  - Border：`rgba(234,240,255,0.12)`
- 字体与层级
  - Font：系统字体优先（Inter/Roboto/苹方/思源黑体）
  - H1：48/56，字重 700
  - H2：32/40，字重 650
  - H3：20/28，字重 650
  - Body：16/26
  - Caption：13/20
- 按钮
  - Primary：Accent 实底 + 轻微外发光（hover 提升亮度/阴影）
  - Secondary：透明底 + 1px 边框（hover 背景加深）
  - Focus：2px 外描边（可访问性）
- 链接
  - 默认：Text Primary，hover 使用 Accent
- 动效
  - 过渡：150–200ms，`ease-out`
  - 入场：仅用于卡片轻微上浮/淡入（避免花哨）

## 4. Page Structure（整体版式）
页面从“定位 → 证据 → 路径 → 背书 → 行动”逐步递进：
1) 先让你相信（专业）
2) 再让你验证（可审计）
3) 最后让你放心（安全优先）
并在中段与底部提供明确 CTA。

## 5. Sections & Components（含文案与结构）

### 5.1 顶部导航（Header）
- 布局：左 Logo + 右侧锚点导航 + 右侧按钮组
- 元素
  - Logo/名称：
    - 名称建议：`Secure-first`（可替换为你的品牌名）
    - Tagline（小字）：`专业 · 可审计 · 安全优先`
  - 导航锚点：价值主张、三步路径、可信背书、开始
  - 按钮：
    - 次按钮：`查看三步路径`
    - 主按钮：`获取评估方案`
- 交互：滚动到对应 section；当前锚点高亮

### 5.2 Hero（首屏主视觉）
- 目标：10 秒内让你知道这是“安全与审计导向”的专业方案，并给出下一步
- 布局：左文案（7/12）+ 右视觉（5/12）
- 右侧视觉建议（避免花哨）：
  - 方案 A：简洁“审计证据链”示意（节点+连线）
  - 方案 B：三项徽章 + 关键指标（例如“可导出审计报告”“最小权限默认”）
- 文案（推荐版本）
  - H1：`把关键系统做成“可证明可信”。`
  - Subhead：`以专业工程实践为底座，用可审计证据与安全优先策略，让你在接入、验证、上线的每一步都可回溯、可检查、可交付。`
  - 主按钮：`获取评估方案`
  - 次按钮：`查看三步路径`
  - 辅助说明（caption）：`默认最小权限 · 可导出审计材料 · 安全边界清晰`

### 5.3 信任条（Trust Bar）
- 位置：Hero 下方紧接一条横向条带，增强专业可信
- 内容建议（不引入具体厂商/认证的前提下）：
  - 文字徽章：`审计友好` `可回溯` `安全优先` `交付可验证`
  - 若你已有认证/合规：在此替换为对应徽章（如 ISO/SOC2 等）

### 5.4 价值支柱（Pillars：专业 / 可审计 / 安全优先）
- 布局：三列卡片（桌面），每张卡包含：标题 + 一句话价值 + “可验证点”列表
- 卡片文案
  1) 专业
     - 一句话：`用工程化方法把复杂问题拆解到可执行。`
     - 可验证点：`清晰边界与责任划分` / `可复用的标准流程` / `可读的交付物结构`
  2) 可审计
     - 一句话：`不是“相信”，而是“能检查”。`
     - 可验证点：`日志与变更可追溯` / `证据可导出可留存` / `关键动作可复现`
  3) 安全优先
     - 一句话：`默认站在风险最小的一侧。`
     - 可验证点：`最小权限默认` / `敏感数据加密与隔离` / `可控的访问与撤销`

### 5.5 三步使用路径（核心重点区）
- 目标：把“怎么用”讲到你能立刻复述给同事
- 布局：横向 3 步卡片 + 顶部标题与一句话
- 标题与导语
  - H2：`三步走：接入 → 验证 → 上线运行`
  - 导语：`每一步都有明确输入、输出与可审计证据。`
- 三步卡片（每张包含：Step 标号、标题、说明、输入/输出、耗时）
  - Step 01｜接入（Define & Integrate）
    - 说明：`确认范围与边界，完成必要接入与基线配置。`
    - 输入：`目标系统/流程范围、权限边界、现有规范`
    - 输出：`范围清单、接入完成证明、基线配置记录`
    - 预计：`1–3 天（视复杂度）`
  - Step 02｜可审计验证（Prove & Audit）
    - 说明：`把关键动作做成可回溯证据链，形成可检查材料。`
    - 输入：`日志/变更/访问记录`
    - 输出：`审计证据包、导出报告、复现步骤`
    - 预计：`3–7 天`
  - Step 03｜安全上线与运行（Operate Securely）
    - 说明：`最小权限、分层防护与持续监测，确保长期可控。`
    - 输入：`验证结果、风险项清单`
    - 输出：`上线清单、运行策略、回滚与应急预案`
    - 预计：`1–2 周（并行推进）`
- 组件细节
  - 每步右上角放“证据”小标签：`可导出` / `可复现` / `可留存`
  - 卡片底部给“你将获得”一行：`交付物：XXX`

### 5.6 可信背书（Auditability & Security Proof）
- 布局：左右分栏
  - 左：可审计证据（列表 + 示例结构）
  - 右：安全优先实践（列表 + 默认策略）
- 文案
  - H2：`你能拿到的，不止是结论，而是证据。`
  - 可审计证据（左）
    - 小标题：`审计证据包（示例）`
    - 列表：
      - `关键事件日志（时间线）`
      - `变更记录与审批链`
      - `访问与权限快照（可对比）`
      - `导出报告（可提交/可留档）`
  - 安全优先实践（右）
    - 小标题：`默认策略（示例）`
    - 列表：
      - `最小权限与可撤销访问`
      - `敏感数据分级与加密`
      - `环境隔离与边界控制`
      - `异常检测与审计告警`

### 5.7 中段 CTA（转化段）
- 目的：给你一个明确“下一步”
- 布局：大块渐变底（Accent 低饱和）+ 左文案右按钮
- 文案
  - 标题：`把你的“安全与审计”从口号变成可交付。`
  - 说明：`告诉我你的范围与目标，我会给你一份三步评估方案与交付清单。`
  - 主按钮：`获取评估方案`
  - 次按钮：`只看三步路径`

### 5.8 FAQ（可选但建议）
- 目的：减少你对“可审计/安全优先”概念的疑虑
- 布局：两列折叠面板（桌面），移动端单列
- 问题建议（不引入新功能）：
  - `“可审计”具体指什么？`
  - `如何保证证据可复现？`
  - `安全优先会不会影响效率？`

### 5.9 页脚（Footer）
- 布局：三列（品牌说明 / 链接 / 联系方式占位）
- 内容
  - 品牌一句话：`专业 · 可审计 · 安全优先`
  - 链接（占位）：`隐私政策` `使用条款` `安全说明`
  - 版权：`© {Year} Your Company. All rights reserved.`
